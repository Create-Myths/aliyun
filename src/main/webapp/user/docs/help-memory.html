云内存存储

什么是云内存存储？

云内存存储是一种支持数据持久化的内存存储服务，使用Memcache协议进行访问，具备高性能和高可用的特点，使用户无需自行搭建及管理memcache服务，减少研发和运维成本。
性能出色——支持高并发，具有高性能读写的显著特点。
快速创建——只需根据业务需求，选择所需内存容量，一键创建，立即使用。
服务高可用——支持双机热备，故障自动切换，不受单点故障的影响。
数据可靠——所有的内存数据均持久化到磁盘，不会因服务故障重启而造成数据丢失，避免数据需要重新预热。
兼容性强——无缝兼容Memcache协议，原有业务可以直接迁移，无需任何修改。
在线扩容——根据实际负载以及内存空间使用情况，可以非常方便地进行在线升级，从而获取更高存储空间与性能，且升级期间，读写服务不会中断。

创建内存存储空间

· 用户登录控制台后，可在控制台内的存存储空间中，进行如下操作步骤：

1、点击管理页面中的“创建内存空间”按钮：

· 2、在弹出的申请页面中，可以输入空间的名称、空间的存储容量大小、是否需要热备、购买数量、计费方式等，并根据您的选择统计费用，页面右侧会显示您的账号余额以及当前所选配置，点击完成创建，将为您生成对应要求的内存存储空间。

扩容

对于已创建成功的内存存储空间，如果其容量已不能满足现有业务的需要，可以对其进行扩容操作。选定需要扩容的空间后，点击存储空间列表上方的“扩容”按钮，将会弹出扩容页面。

删除

您也可以删除不需要的内存存储空间，删除时，只需在选定目标内存存储空间后，点击存储空间列表上方的删除按钮。

FAQ

云内存存储的安全性怎么样？

· 访问安全性
UMem实例是“仅内网访问且按账户隔离”的，因此仅有同一账户的云主机能够对UMem实例进行访问。
· 数据安全性
所有的内存数据均持久化到磁盘，不会因服务故障重启而造成数据丢失，避免数据需要重新预热。


云内存存储支持哪些协议？是否为原生协议？

· 目前，UMem仅支持了原生Memcache协议，后续会考虑支持更多的协议种类。
· 由于支持原生Memcache协议，所以现有使用Memcache的服务如果需要迁移至UMem，可以不做任何修改，直接迁移。


如何使云内存存储提供的多个ip？

每个实例都会提供两个IP进行访问。这两个IP都可以对UMem进行访问，分布在不同的接入服务上，其作用在于，当其中一个IP无法正常访问时，仍有另一个IP可用，不会完全中止服务。因此，应用程序可以增加一个容灾切换的逻辑处理：将访问的IP列表设置好，默认访问其中的一个IP，当该IP无法访问时，自动切换到另一个IP继续业务

云内存存储如何确保存储服务的高可用？

在创建UMem内存存储空间时，可以选择“有热备”的类型，该类型的存储空间将采用双机热备的方式来确保可用性，即有主从两个存储节点，并且实时同步保证数据的一致性，如果主节点发生宕机，系统会自动切换到从节点上，继续提供读写服务。

如果存储空间容量不足了怎么办？

如果存储空间容量不足，可以对其进行扩容，并且该扩容操作是支持在线方式的，一键操作即可完成，扩容期间不会中断业务，读写操作仍然可以正常进行。
